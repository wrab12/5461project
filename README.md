## 🎯 Project Overview

This project conducted a systematic benchmark test of the COMMOT (COMMunication analysis by Optimal Transport) method, validating the accuracy of COMMOT's cell-cell communication inference through synthetic and real datasets.

## 🔬 Key Findings

### 1. COMMOT vs Ground-truth Benchmark

**Feature Correlation: 0.14** (low correlation)
- COMMOT-inferred features are almost uncorrelated with simple distance-decay ground-truth
- Indicates COMMOT uses a unique optimal transport inference mechanism

**Performance Retention: 102%** (COMMOT even slightly better)
- Ground-truth average ARI: 0.8304
- COMMOT average ARI: 0.8468
- Proves COMMOT is not simply copying spatial distance

### 2. COMMOT Has Advantages in Complex Structures

| Dataset | Ground-truth ARI | COMMOT ARI | Improvement |
|---------|-----------------|------------|-------------|
| Tissue | 0.703 | **0.941** | +34% |
| Radial | 0.630 | **0.889** | +41% |
| Striped | **1.000** | 0.844 | -16% |
| Gradient | **0.990** | 0.713 | -28% |

**Conclusion:**
- Complex spatial structures (Tissue, Radial): COMMOT significantly better
- Simple regular structures (Striped, Gradient): Simple methods sufficient

### 3. Impact of Spatial Patterns on CCC Clustering

**Successful Patterns (ARI > 0.7):**
- Directional and structured: Tissue, Radial, Striped
- Characteristics: Clear spatial organization and communication direction

**Failed Patterns (ARI ≈ 0):**
- Clustered (isolated clusters): Cannot improve regardless of parameter adjustment
- Reason: All clusters have too similar CCC patterns (all "high reception in center, high sending at edges")

### 4. K-means Performs Best

On COMMOT features:
- K-means average ARI: 0.847
- Louvain average ARI: 0.320
- Leiden average ARI: 0.230

Indicates COMMOT-extracted CCC feature space is relatively regular, suitable for simple clustering algorithms.

### 5. Important Findings from Real Data Analysis ⭐

**Problem:** Direct clustering on raw CCC features produces "salt-and-pepper" chaotic results

**Reasons:**
- CCC features are high-dimensional and continuous, with large value range differences across L-R pairs
- Without feature standardization, large signals dominate clustering results
- Without spatial constraints, algorithm doesn't know spatial proximity of cells

**Solutions:**
1. **Feature Preprocessing**:
   - Variance filtering → Log transformation → Standardization → PCA dimensionality reduction (598 dims → 30 dims)
   
2. **Three Clustering Strategies**:
   - Strategy A: CCC features only (after PCA)
   - Strategy B: CCC + Spatial coordinates (0.2 weight) - **Best**
   - Strategy C: Raw features (baseline comparison)

**Results:**
- Strategy B produced more spatially continuous partitions that better reflect tissue structure
- Proves CCC signals are continuous and high-dimensional, not discrete domains
- This is the expected property of real tissue data

**Scientific Conclusion:**
> "The real data showed spatially varying sender and receiver communication signals. However, clustering based only on raw CCC features produced mixed spatial patterns, suggesting that CCC signals are continuous and high-dimensional rather than sharply separated into discrete spatial domains. After proper feature preprocessing (log transformation, standardization, PCA) and optional spatial regularization, clustering stability improved significantly, producing more spatially coherent patterns that better reflect tissue organization."

## 📝 Clustered Case Study

Preserved 3 versions of Clustered fix attempts as a failure case study:

1. **Original Version**: Ratio 2.83x, ARI=0.006 (cluster overlap)
2. **Fix V1**: Ratio 6.82x, ARI=0.001 (cluster isolation)
3. **Optimized V2**: Ratio 3.98x, ARI=-0.001 (balanced but still failed)

**Lessons:**
- Not all spatial patterns are suitable for CCC-based clustering
- Isolated cluster patterns lack directionality, CCC features cannot distinguish
- Successfully replaced with Radial (radial structure), ARI improved to 0.889

## 🔍 Methodological Contributions

### Best Practices for Real Data Analysis

**Not Recommended:**
```python
# ❌ Direct clustering on raw features
features = np.hstack([sender, receiver])
sc.pp.neighbors(adata, use_rep='X')
sc.tl.leiden(adata)
```

**Recommended:**
```python
# ✅ Complete preprocessing pipeline
# 1. Variance filtering
vt = VarianceThreshold(threshold=1e-8)
X = vt.fit_transform(X)

# 2. Log transformation
X = np.log1p(X)

# 3. Standardization
X = StandardScaler().fit_transform(X)

# 4. PCA dimensionality reduction
pca = PCA(n_components=30)
X_pca = pca.fit_transform(X)

# 5. Optional: Add spatial constraints
coords_scaled = StandardScaler().fit_transform(coords)
X_joint = np.hstack([X_pca, 0.2 * coords_scaled])

# 6. Clustering
sc.pp.neighbors(adata, use_rep='X')
sc.tl.leiden(adata, resolution=0.3)
```
